# 本地音乐管理功能文档

## 功能概述

本地音乐管理功能是一个完整的音乐文件管理系统，允许用户浏览本地音乐文件、搜索在线匹配、选择最佳匹配、下载音乐文件，并提供任务管理和持久化存储功能。

## 主要功能

### 1. 文件夹选择与管理

#### 浏览文件夹
- 使用 `window.showDirectoryPicker()` API 选择本地文件夹
- 支持递归遍历文件夹结构
- 自动识别支持的音频格式：`.mp3`, `.flac`, `.m4a`, `.wav`
- 使用 jsmediatags 库读取音频文件的元数据（歌曲名、歌手等）

#### 文件夹标识
- 使用文件夹名称或路径作为唯一标识
- 通过哈希算法计算 `library_id`
- 支持多个不同的文件夹管理

#### 库文件管理
- 库文件存储在 `libraries/` 目录下
- 每个文件夹对应一个 JSON 格式的库文件
- 库文件包含：歌曲信息、搜索结果、最佳匹配、下载状态等

### 2. 歌曲信息管理

#### 歌曲信息展示
- **歌曲列表**：左侧面板显示所有歌曲，支持多种排序方式
  - 按文件名排序
  - 按状态排序
  - 按歌手排序
  - 按歌曲名排序

- **歌曲详情**：右侧面板显示选中歌曲的详细信息
  - 歌曲名称
  - 歌手
  - 时长
  - 状态
  - 文件名
  - 搜索结果（QQ音乐、网易云音乐）

#### 状态灯系统
- **红色**：未搜索
- **绿色**：已搜索
- **蓝色**：已选择最佳匹配
- **紫色**：已下载
- **黄色**：搜索失效、下载错误或其他错误

### 3. 搜索功能

#### 搜索控制
- **开始匹配**：启动搜索任务，从第一个需要搜索的歌曲开始
- **暂停**：暂停当前搜索任务
- **继续**：继续已暂停的搜索任务
- **线程数选择**：支持 1-10 线程并发搜索

#### 搜索结果
- 同时搜索 QQ音乐 和 网易云音乐
- 每个来源显示前 3 个结果
- 搜索结果包含：
  - 歌曲名称
  - 歌手
  - 专辑
  - 时长
  - 文件大小
  - 文件格式
  - 试听链接
  - 下载链接

#### 搜索结果排序
- 完美匹配优先（歌曲名和歌手都匹配）
- 歌曲名匹配次之
- 智能排序算法确保最相关的结果排在前面

### 4. 最佳匹配选择

#### 设为最佳
- 点击搜索结果中的"设为最佳"按钮
- 系统保存选择的歌曲信息
- 歌曲状态变为"已选择"（蓝色灯）

#### 取消选择
- 点击已选择歌曲的"取消选择"按钮
- 系统清除最佳匹配
- 歌曲状态恢复为"已搜索"（绿色灯）

#### 最佳匹配标识
- 已选择的搜索结果会显示绿色背景
- 在歌曲列表中显示蓝色状态灯

### 5. 下载功能

#### 下载全部最佳匹配
- 批量下载所有已选择最佳匹配的歌曲
- 自动处理下载失败情况
- 支持断点续传

#### 下载状态管理
- 实时显示下载进度
- 下载完成后自动更新歌曲状态
- 支持重置已下载状态

#### 下载失败处理
- 网易云音乐下载链接失效时自动重新搜索
- 智能匹配之前的最佳选择（相同来源、歌曲名、歌手、文件后缀）
- 重新搜索后仍然失败则标记为"搜索失效"

### 6. 时长管理

#### 自动获取时长
- 使用 HTML5 Audio 元素获取音频时长
- 格式化为 `MM:SS` 格式
- 超时处理：5秒超时保护

#### 刷新时长
- 手动触发时长获取功能
- 只更新缺失时长的歌曲
- 保留现有的搜索进度和状态

### 7. 任务管理

#### 任务状态查看
- **搜索任务**：显示当前搜索任务的状态、线程数、进度索引
- **下载任务**：显示当前下载任务的状态、总任务数、已完成数

#### 任务控制
- **重新开始搜索**：重新启动搜索任务
- **重新开始下载**：重新启动下载任务

#### 持久化存储
- **保存任务状态**：将当前任务状态保存到浏览器本地存储
  - 保存库 ID
  - 保存时间戳
  - 保存所有歌曲信息
  
- **加载任务状态**：从浏览器本地存储加载之前保存的任务状态
  - 恢复搜索进度
  - 恢复最佳匹配选择
  - 恢复歌曲状态

#### 库文件管理
- **加载库文件**：查看所有可用的库文件
  - 显示库 ID、名称、修改时间
  - 按修改时间排序（最新的在前）
  - 点击即可加载对应的库文件

### 8. 播放功能

#### 本地歌曲播放
- 使用 Blob URL 技术播放本地文件
- 在歌曲详情中显示播放按钮
- 支持暂停、继续、音量控制

#### 在线歌曲试听
- 点击搜索结果中的"试听"按钮
- 直接播放在线音频
- 用于验证搜索结果的正确性

#### 播放器控制
- 底部固定播放器
- 显示当前播放的歌曲信息
- 支持完整的 HTML5 Audio 控件

### 9. UI 交互

#### 响应式布局
- 左右分栏设计
- 可拖拽调整面板宽度
- 支持不同屏幕尺寸

#### 实时更新
- 搜索进度实时显示
- 下载进度实时更新
- 状态灯实时变化

#### 进度统计
- 显示搜索进度（已搜索/总数/百分比）
- 显示选择进度（已选择/总数/百分比）
- 下载状态实时更新

### 10. 数据持久化

#### 自动加载
- 页面加载时自动加载上次使用的库
- 使用 localStorage 保存库 ID
- 清除失效的 Blob URL

#### 库文件更新
- 智能更新现有库文件
- 保留搜索进度和最佳匹配
- 更新音频时长等新信息

#### 状态保留
- 保留所有有效的歌曲状态
- 包括搜索失效等特殊状态
- 确保重启后状态不丢失

### 11. 错误处理

#### 未识别文件
- 显示不支持的文件列表
- 支持的格式：`.mp3`, `.flac`, `.m4a`, `.wav`
- 提供文件路径信息

#### 搜索失效处理
- 自动识别搜索失效状态
- 标记为黄色状态灯
- 支持重新搜索

#### 下载错误处理
- 自动重试机制
- 智能匹配最佳结果
- 失败后标记为搜索失效

### 12. 其他功能

#### 重置已下载
- 将所有已下载歌曲重置为未下载状态
- 需要用户确认
- 批量处理

#### 显示未识别文件
- 查看不支持的文件列表
- 显示文件数量
- 帮助用户了解文件情况

## 技术实现

### 前端技术
- **HTML5**：页面结构
- **CSS3**：样式和布局
- **JavaScript**：交互逻辑
- **File API**：文件系统访问
- **Blob URL**：本地文件播放
- **localStorage**：本地数据存储

### 后端技术
- **FastAPI**：Web 框架
- **Pydantic**：数据验证
- **JSON**：数据存储格式
- **Background Tasks**：异步任务处理

### 数据模型
- **LocalLibrary**：库数据模型
- **LocalSong**：歌曲数据模型
- **CreateLibraryRequest**：创建库请求模型
- **SelectChoiceRequest**：选择最佳匹配请求模型

## API 接口

### 库管理接口
- `POST /api/local/create-library`：创建或更新库
- `GET /api/local/library-state`：获取库状态
- `GET /api/local/library-list`：获取库文件列表

### 搜索控制接口
- `POST /api/local/search-control`：搜索任务控制
- `GET /api/local/task-status`：获取搜索任务状态

### 下载控制接口
- `POST /api/local/download-control`：下载任务控制
- `GET /api/local/download-status`：获取下载任务状态

### 选择接口
- `POST /api/local/select-choice`：选择最佳匹配

### 重置接口
- `POST /api/local/reset-downloads`：重置已下载状态

## 使用流程

### 基本使用流程
1. 点击"浏览文件夹"选择本地音乐文件夹
2. 系统自动识别音频文件并创建库
3. 点击"开始匹配"启动搜索任务
4. 查看搜索结果并选择最佳匹配
5. 点击"下载全部最佳匹配"下载音乐
6. 查看下载进度和状态

### 任务恢复流程
1. 点击"任务管理"按钮
2. 选择"加载库文件"或"加载任务状态"
3. 选择要恢复的库文件或任务状态
4. 系统自动恢复之前的进度和选择

### 时长更新流程
1. 点击"刷新时长"按钮
2. 选择相同的音乐文件夹
3. 系统自动获取音频时长
4. 保留现有的搜索进度和状态

## 注意事项

1. **浏览器兼容性**：需要支持 File API 的现代浏览器
2. **文件夹权限**：需要授予文件夹访问权限
3. **网络连接**：搜索和下载功能需要网络连接
4. **存储空间**：确保有足够的磁盘空间存储下载的音乐文件
5. **线程设置**：根据网络情况调整线程数，避免过载

## 总结

本地音乐管理功能提供了一个完整的音乐文件管理解决方案，集成了文件浏览、在线搜索、智能匹配、批量下载、任务管理和持久化存储等功能。通过直观的界面和强大的功能，用户可以高效地管理本地音乐文件，并轻松获取高质量的在线音乐资源。
